## 1. 摘要

AIggs 是第一款完全通过 AI 工具进行游戏的区块链游戏。玩家不需要打开任何 App 或访问网站——他们的 AI 助手（Claude、GPT 或任何兼容 MCP 协议的工具）会代替玩家加入农场、查看产量、执行偷蛋行动、管理代币兑换。

玩法极简：母鸡每 8 小时自动产出 EGGS。EGGS 在 Base L2 上兑换为 `$AIGG` 代币。`$AIGG` 可通过全球供应商网络兑换真实鸡蛋，或在公开市场交易。

AIggs 在结构上有三点根本性的不同：

1. **AI 原生分发。** 游戏通过 AI 工具生态触达用户，而非依赖应用商店。可寻址的分发渠道是全球所有 Claude 和 ChatGPT 的活跃用户。
2. **真实世界价值锚定。** `$AIGG` 的价格底部由实物鸡蛋兑换价值支撑。只要鸡蛋有市场价值，`$AIGG` 就不会归零。
3. **协议自有永久流动性。** 所有法币收入的 60% 将永久注入流动性池。LP 代币销毁至 `0x000...`，任何人（包括团队）无法取回。

## 2. 背景与机遇

### 2.1 区块链游戏需要新的信任架构

Web3 游戏的第一波浪潮验证了核心命题：玩家愿意为链上资产付费，也愿意在游戏中积累真实价值。但早期设计将代币经济与流动性控制混为一谈，导致整个行业沉淀了大量具有链上资产需求却缺乏合适产品的用户。

这是一个机遇，而非障碍。真正可持续的链上游戏需要满足三个条件：无法被团队操控的流动性结构、独立于投机之外的真实世界需求、以及超越技术门槛的用户体验。AIggs 在设计上同时满足这三点。

### 2.2 AI 工具正在寻找经济层

全球数以亿计的用户每天使用 AI 工具写作、编程、搜索。这些交互处理的是信息，却没有为用户创造任何持久的经济积累。用户每天与 AI 互动，每次对话结束后一无所有。

AI 工具拥有前所未有的用户黏性和使用频率，却尚未具备跨会话管理持续资产的能力。这是一个结构性空缺——AIggs 正是为填补这个空缺而设计的。将游戏状态嵌入 AI 工具记忆，使每一次 AI 对话都可以成为一次农场管理行动，让用户在日常 AI 使用中自然积累链上价值。

### 2.3 消费级真实资产游戏的蓝海市场

"RWA"（真实世界资产）是 DeFi 领域最强的叙事之一，但现有实践大多面向机构资本：代币化国债、房地产、大宗商品基金。没有任何消费级产品成功地将数字游戏经济与实物商品兑付体系连接起来。

市场空间巨大。全球鸡蛋年消费规模超过 3200 亿美元。将数字游戏行为与这条供应链连接的代币化层，至今尚未被建立。AIggs 选择鸡蛋作为切入点，不仅因为其市场规模，更因为它是全球最普遍、文化接受度最高的食品品类之一。

## 4. 核心玩法

### 4.1 农场与产蛋

每位新玩家注册即获赠一只免费母鸡，无需钱包，无需付费。

| 参数 | 数值 | 说明 |
|---|---|---|
| 初始母鸡 | 1只（免费） | 无需钱包，零门槛 |
| 产蛋速率 | 每只鸡每8小时产1枚 | 每天每只鸡产3枚 |
| 母鸡寿命 | 30天 | 到期停产，需补充 |
| 仓库初始容量 | 30枚 | 可通过升级扩容 |
| 仓库满后 | 停止产蛋，EGGS不销毁 | 用户不会因遗忘而损失 |
| 每日兑换上限 | 2枚 `$AIGG`（初始） | 随账号等级提升 |

**设计原则——满仓停产而非销毁：**
仓库满时，系统暂停产蛋而不是销毁鸡蛋。这会形成"回来收蛋"的主动拉力，而非惩罚不活跃的玩家。"我要去取蛋"和"我的蛋烂掉了"——这两种心理体验截然不同，对留存率的影响也有本质差异。

### 4.2 身份系统：农场码

每位玩家注册时获得唯一农场码（如 `farm-x7k2-9p3m`）。这串代码存储在 AI 工具的记忆中，后续每次操作自动携带。

```
用户 → AI工具：  「帮我加入AIggs」
AI工具 → API：   POST /join
API → AI工具：   { farm_code: "farm-x7k2-9p3m", name: "旭日农场" }
AI工具 → 用户：  「你的农场已建好，农场码已记住。」
```

从此以后，玩家无需管理私钥、钱包地址或登录凭证。AI 处理所有身份识别。高价值操作（兑换 `$AIGG`、实物兑付）需要在第3周引入手机号二次验证。

### 4.3 每日早报

每天，AI 工具主动推送结构化早报：

```
🌅 旭日农场 · Day 14
──────────────────────
昨日收成：  +3枚 EGGS（1只鸡 × 3次产蛋）
⚠ 昨晚被偷：-6枚（夜半农场 · 22:47）
当前库存：  24 / 30 枚
今日汇率：  30 EGGS = 1 $AIGG
$AIGG参考价：$0.10 U
──────────────────────
解锁进度：[████████░░] 24/30 → 偷蛋资格
```

早报是 AIggs 最核心的留存界面。具体的被盗信息——农场名称、时间戳、数量——是刻意设计的。有了具体对手和具体时间，玩家会产生真实的情绪反应（报仇的冲动、加强防守的意愿），而模糊的"昨晚被偷了"则不会。

## 5. 偷蛋机制与博弈论

偷蛋是 AIggs 的社交引擎，将一款被动放置游戏转变为主动竞争游戏，是驱动日活留存的核心钩子。

### 5.1 基础规则

| 规则 | 数值 |
|---|---|
| 每日偷蛋次数 | 每人2次 |
| 新用户保护期 | 注册后24小时内不可被偷，也不能偷人 |
| 目标最低保护库存 | 15枚（低于此值不可被偷） |
| 同目标冷却 | 成功偷取后24小时内不能再次偷同一目标 |
| 失败惩罚 | 无——失败只消耗次数 |

### 5.2 三种结果

| 结果 | 概率 | 发生了什么 | 玩家感受 |
|---|---|---|---|
| 🎉 大丰收 | 15% | 偷走目标仓库的 20–25% | 兴奋，有分享冲动 |
| ✅ 成功 | 45% | 偷走目标仓库的 10–15% | 满足，继续参与 |
| 😅 扑空 | 40% | 什么都没偷到，对方也不知道 | 轻微失落，不至于气馁 |

扑空率刻意设置为40%。失败率过低会让偷蛋变得机械；失败率过高会打击积极性。"对方不知道自己被尝试偷过"这一信息不对称设计，有效降低了骚扰和恶意行为。

### 5.3 宿敌系统

当两名玩家互相偷蛋达到3次或以上，系统自动将其标记为**宿敌**。宿敌关系：

- 永久存在，无法解除
- 在双方每日早报中持续体现
- AI 每日主动提及："你的宿敌夜半农场，18小时前刚偷过你。"

宿敌关系是 AIggs 的社交图谱。它创造了持续回来的理由——不是为了完成任务，而是为了了结一笔旧账。

### 5.4 留存模型

偷蛋机制的设计基于一个核心观察：**被偷会产生情绪；情绪驱动回访。**

```
玩家夜间被偷
      ↓
早报：具体的偷蛋者名字 + 时间 + 数量
      ↓
情绪反应（愤怒、不甘、竞争冲动）
      ↓
玩家当天回来报仇或加强防守
      ↓
产生一次有效的活跃行为
```

内部目标：被偷玩家当天返回率 > 50%。这是第3–4周最核心的验收指标。

## 6. 代币经济

### 6.1 双资产体系

AIggs 运行两种独立资产：

| 资产 | 类型 | 存储位置 | 功能 |
|---|---|---|---|
| EGGS | 链下积分 | 游戏服务器 | 游戏内货币；不可转移 |
| `$AIGG` | ERC-20 代币 | Base L2 | 链上价值；可交易；可兑付 |

EGGS 不是代币，无法在钱包间转移，也无法对外交易。这种分离设计从根本上消除了早期流动性攻击向量，同时确保 `$AIGG` 的稀缺性得到保护。

### 6.2 EGGS 流动

**来源：**

| 来源 | 速率 | 说明 |
|---|---|---|
| 母鸡产蛋（主要） | 每只每天3枚 | 服务端定时，无需用户触发 |
| 每日签到奖励 | +1枚 | 当天首次查询触发 |
| 邀请好友分成 | 被邀请者产出的10% | 永久分成，第5周引入 |
| 成功偷蛋 | 偷到多少算多少 | 攻击方收益 = 防守方损失 |

**消耗：**

| 消耗场景 | 成本 | 说明 |
|---|---|---|
| 兑换 `$AIGG`（主要） | 30枚 = 1 `$AIGG` | 主要消耗出口 |
| 孵化新母鸡 | 6枚/只 | 扩大产能 |
| 购买看门狗 | 5枚/天 | 第5周引入 |

### 6.3 `$AIGG` 供应与分配

**总供应量：10亿枚 `$AIGG`**

| 分配 | 比例 | 数量 | 锁仓规则 |
|---|---|---|---|
| 玩家（养殖赚取） | 75% | 7.5亿枚 | 通过游戏行为赚取 |
| 团队 | 20% | 2亿枚 | 2年锁仓；按季度解锁 |
| 空投（前1万名用户） | 5% | 5000万枚 | TGE解锁 |

### 6.4 收入分配

所有法币和加密货币收入通过链上固定比例分配。合约部署后不可更改。

```
用户付款（法币或加密货币）
          │
    ┌─────┴─────┐
   60%          40%
永久流动性        国库
    │        ┌───┴──────────┐
50% $AIGG   45% 运营      30% 稳定基金
+ 50% USDC  15% 生态建设
→ LP代币    10% 团队
→ 销毁至
  0x000...
```

**60% 永久流动性规则**是整个信任架构的核心。每一笔购买都永久加深 `$AIGG/USDC` 流动性池。LP 代币在交易时即刻发送至销毁地址。团队成员、治理投票、任何紧急机制都无法取回这部分资金。

### 6.5 价格底部机制

`$AIGG` 存在一个由实物鸡蛋兑付价值支撑的软性价格底部：

> **1 `$AIGG` ≈ 10 枚真实鸡蛋**

这个价格底部不是团队人为维护的——它由套利行为自动执行。如果 `$AIGG` 交易价格低于兑付价值，任何持有者都可以低价购入代币、兑换鸡蛋、按市价卖出并获利。这一自我修正机制全自动运行，无需任何人为干预。

其他通缩压力来源：

- **道具购买：** 游戏内所有消费（购鸡、防盗装备、升级）永久销毁 `$AIGG`
- **流动性锁定：** 60% 法币收入持续买入并永久锁仓 `$AIGG`
- **供应上限：** 固定10亿枚总量，TGE后无增发功能

### 6.6 汇率机制

| 阶段 | 汇率 | 机制 |
|---|---|---|
| 第1–6周 | 30 EGGS = 1 `$AIGG`（固定） | 简单直观，易于传播 |
| 第7周起 | 动态汇率（每日最大波动±10枚） | 供需比驱动自动调整 |

每日汇率在所有玩家的早报中公示，玩家可以据此规划最佳兑换时机。汇率历史数据在 Web 界面上线后可在链上查询。

## 7. 供应商网络与真实世界资产

### 7.1 运作方式

供应商质押 `$AIGG` 加入网络。希望兑付的玩家提交订单。供应商完成订单（实物鸡蛋配送）并赚取服务费。协议负责撮合匹配和争议处理。

质押的 `$AIGG` 在供应商参与期间全程锁定。这为代币创造了随网络规模增长而持续扩大的结构性需求。

长期愿景是建立品牌合作层，使 `$AIGG` 成为可在整个食品生态中兑付的忠诚度货币——以鸡蛋为起点，逐步扩展至蛋白质、餐厅和外卖平台。

## 8. AI 原生架构

### 8.1 游戏即 Skill：全新的分发范式

传统游戏的用户获取路径：

```
用户 → 发现App → 下载 → 注册 → 教学关卡 → 开始玩
```

AIggs 的路径：

```
用户 → 对AI说「帮我养鸡」→ 开始玩
```

AIggs 作为一个可安装的 **Skill（技能包）** 发布。用户安装 AIggs Skill 后，Claude、ChatGPT 等 AI 工具即获得完整的农场管理能力。游戏不存在于任何 App Store——它存在于 AI 工具的能力体系之中。

这是一种在结构上难以被复制的分发优势。AIggs 不需要争夺应用商店排名或广告位，也不需要说服用户安装新 App。它通过 AI 工具生态自然传播——每一个安装了 AIggs Skill 的 AI 用户，都成为游戏的原生触点。

**Skill 化带来的三重优势：**

- **零摩擦入门：** 用户在已有的 AI 对话界面中直接开始游戏，无需切换场景
- **主动推送：** AI 工具可在最优时机主动提醒用户（早报、满仓警告、偷蛋建议），而不是等待用户主动打开 App
- **跨平台同步：** 同一个农场可通过不同 AI 平台访问，用户不会因为换了工具而失去游戏进度

### 8.2 为什么"AI 原生"至关重要

大多数区块链游戏将 AI 视为一项功能——客服聊天机器人，或生成式内容工具。AIggs 颠倒了这一关系：**AI 工具就是主要交互界面。** 游戏从设计之初就是为通过对话式 AI 进行游玩而构建的。

可寻址的分发渠道是全球所有 Claude 和 ChatGPT 的活跃用户——这是任何传统游戏发行渠道都无法比拟的规模。

### 8.3 MCP 集成

AIggs 作为原生 MCP（模型上下文协议）服务器发布。任何兼容 MCP 协议的 AI 工具安装 AIggs 服务器后，即可将所有游戏功能作为原生工具调用：

```json
{
  "tools": [
    "aiggs_join",
    "aiggs_status",
    "aiggs_morning_report",
    "aiggs_steal",
    "aiggs_convert",
    "aiggs_neighbors"
  ]

```

安装后，AI 工具将自动：
- 跨会话持久保存农场码
- 主动推送每日早报
- 在最优时机建议偷蛋目标
- 仓库即将满仓时发出提醒

### 8.4 多平台兼容

| 平台 | 接入方式 | 农场码持久化 |
|---|---|---|
| Claude（Anthropic） | Cowork Skill / MCP 服务器 | 保存在 Skill 配置中 |
