## 1. 摘要

AIggs 是第一款真正意义上 AI 原生的链上游戏。"AI 原生"在这里不只是指通过 AI 工具来玩游戏——而是指这款游戏的分发、运营、平衡调整、社区治理与产品决策，全部由 AI 主导执行。

玩法极简：母鸡每 8 小时自动产出 EGGS。EGGS 在 Base 上兑换为 `$AIGG` 代币，可在公开市场交易。玩家可以偷取邻居的蛋来加速积累，也可以防守自己的仓库。

AIggs 在结构上有三点根本性的不同：

1. **游戏即 Skill。** AIggs 作为 AI 工具技能包发布，用户只需对 AI 说一句话即可开始游戏。可寻址的分发渠道是全球所有 Claude 和 ChatGPT 的活跃用户。
2. **AI 游戏操作系统。** 设计、运营、决策三类专用 AI Agent 分别负责游戏平衡调整、用户运营与社区管理、产品迭代决策，由 Meta-AI Director 统一协调。人工监督以最小化方式保留，仅覆盖熔断审批与合规边界。
3. **协议自有永久流动性。** 所有收入的 60% 永久注入流动性池，LP 代币销毁至零地址，任何人无法取回。

AIggs 的游戏机制刻意保持简单。简单不是约束，而是核心优势——它使 AI 治理的"奖励函数"清晰可量化：留存率、产蛋量、偷蛋活跃度、代币兑换量。这让 AIggs 成为第一个可以真正交给 AI 完整运营的消费级链上产品。

## 2. 背景与机遇

### 2.1 区块链游戏需要新的信任架构

Web3 游戏的第一波浪潮验证了核心命题：玩家愿意为链上资产付费，也愿意在游戏中积累真实价值。但早期设计将代币经济与流动性控制混为一谈，团队可以在任意时刻取走流动性，社区对此毫无制衡手段。

真正可持续的链上游戏需要同时满足三个条件：无法被任何人操控的流动性结构、由玩家行为驱动而非投机支撑的代币需求、以及超越技术门槛的用户体验。AIggs 在设计上同时满足这三点。

### 2.2 AI 工具正在寻找经济层

全球数以亿计的用户每天使用 AI 工具写作、编程、搜索。这些交互处理的是信息，却没有为用户创造任何持久的经济积累。

AI 工具拥有前所未有的用户黏性，却尚未具备跨会话管理持续资产的能力。这是一个结构性空缺——AIggs 正是为填补这个空缺而设计的。将游戏状态嵌入 AI 工具记忆，使每一次 AI 对话都可以成为一次农场管理行动，让用户在日常 AI 使用中自然积累链上价值。

### 2.3 游戏运营的 AI 革命尚未发生

游戏行业每年在运营、内容更新、用户服务和平衡调整上投入巨大的人力成本。但这些工作的大部分本质上是数据驱动的决策——分析玩家行为、识别异常、调整参数、输出内容。

AI 已经具备接管这些工作的能力，但游戏行业还没有一个消费级产品真正建立起这套运营架构。AIggs 将成为第一个。

## 4. 核心玩法

### 4.1 农场与产蛋

每位新玩家注册即获赠一只免费母鸡，无需钱包，无需付费。

| 参数 | 数值 | 说明 |
|---|---|---|
| 初始母鸡 | 1只（免费） | 无需钱包，零门槛 |
| 产蛋速率 | 每只鸡每8小时产1枚 | 每天每只鸡产3枚 |
| 母鸡寿命 | 30天 | 到期停产，需补充 |
| 仓库初始容量 | 30枚 | 可通过升级扩容 |
| 仓库满后 | 停止产蛋，EGGS不销毁 | 用户不会因遗忘而损失 |

仓库满时，系统暂停产蛋而不是销毁鸡蛋。这会形成"回来收蛋"的主动拉力，而非惩罚不活跃的玩家。

### 4.2 身份系统：农场码

每位玩家注册时获得唯一农场码（如 `farm-x7k2-9p3m`）。这串代码存储在 AI 工具的记忆中，后续每次操作自动携带。

```
用户 → AI工具：  「帮我加入AIggs」
AI工具 → API：   POST /join
API → AI工具：   { farm_code: "farm-x7k2-9p3m", name: "旭日农场" }
AI工具 → 用户：  「你的农场已建好，农场码已记住。」
```

玩家无需管理私钥、钱包地址或登录凭证。AI 处理所有身份识别。高价值操作（兑换 `$AIGG`）通过手机号二次验证完成。

### 4.3 每日早报

每天，AI 工具主动推送结构化早报：

```
🌅 旭日农场 · Day 14
──────────────────────
昨日收成：  +3枚 EGGS（1只鸡 × 3次产蛋）
⚠ 昨晚被偷：-6枚（夜半农场 · 22:47）
当前库存：  24 / 30 枚
今日汇率：  30 EGGS = 1 $AIGG
$AIGG参考价：$0.10 U
──────────────────────
解锁进度：[████████░░] 24/30 → 偷蛋资格
```

早报是 AIggs 最核心的留存界面。具体的被盗信息——农场名称、时间戳、数量——创造了具体的竞争关系和回访动力。

## 5. 偷蛋机制与博弈论

偷蛋是 AIggs 的社交引擎，将一款被动放置游戏转变为主动竞争游戏。

### 5.1 基础规则

| 规则 | 数值 |
|---|---|
| 每日偷蛋次数 | 每人2次 |
| 每次偷蛋成本 | 3枚 EGGS（无论成功与否） |
| 新用户保护期 | 注册后24小时内不可被偷，也不能偷人 |
| 目标最低保护库存 | 15枚（低于此值不可被偷） |
| 同目标冷却 | 成功偷取后24小时内不能再次偷同一目标 |

偷蛋需要消耗自己的 EGGS，使每一次出手都是真实的策略决策。失败不会通知目标方，信息不对称保持了游戏张力。

### 5.2 三种结果

| 结果 | 概率 | 发生了什么 |
|---|---|---|
| 🎉 大丰收 | 20% | 偷走目标仓库的 20–25% |
| ✅ 成功 | 55% | 偷走目标仓库的 10–15% |
| 😅 扑空 | 25% | 什么都没偷到，对方也不知道 |

## 6. 代币经济

### 6.1 双资产体系

| 资产 | 类型 | 存储位置 | 功能 |
|---|---|---|---|
| EGGS | 链下积分 | 游戏服务器 | 游戏内货币；不可转移 |
| `$AIGG` | ERC-20 代币 | Base | 链上价值；可交易 |

EGGS 不是代币，无法在钱包间转移，也无法对外交易。这种分离设计从根本上消除了早期流动性攻击向量。

### 6.2 EGGS 流动

**来源：**

| 来源 | 速率 |
|---|---|
| 母鸡产蛋 | 每只每天3枚 |
| 每日签到奖励 | +1枚 |
| 邀请好友分成 | 被邀请者产出的10%（永久） |
| 成功偷蛋 | 偷到多少算多少 |

**消耗：**

| 消耗场景 | 成本 |
|---|---|
| 兑换 `$AIGG` | 30枚 = 1 `$AIGG` |
| 孵化新母鸡 | 6枚/只 |
| 偷蛋行动 | 3枚/次（无论成败） |
| 购买防盗道具 | 5枚/天 |

### 6.3 `$AIGG` 供应与分配

**总供应量：10亿枚 `$AIGG`**

| 分配 | 比例 | 数量 | 锁仓规则 |
|---|---|---|---|
| 玩家（游戏赚取） | 75% | 7.5亿枚 | 通过游戏行为赚取 |
| 生态与社区 | 10% | 1亿枚 | 按里程碑分批释放 |

### 6.4 收入分配

所有收入通过链上固定比例分配，合约部署后不可更改。

```
用户付款
    │
┌───┴───┐
60%     40%
永久流动性  国库（运营与生态建设）
    │
50% $AIGG + 50% USDC
→ LP代币 → 销毁至 0x000...
```

每一笔购买都永久加深 `$AIGG/USDC` 流动性池。LP 代币发送至销毁地址后，任何人无法取回。

### 6.5 通缩机制

- **游戏内消费：** 购鸡、防盗道具、升级等永久销毁 `$AIGG`
- **流动性锁定：** 60% 收入持续买入并永久锁仓 `$AIGG`
- **固定供应上限：** 10亿枚总量，TGE 后无增发功能
- **兑换需求：** EGGS → `$AIGG` 产生持续购买压力

## 7. AI 原生架构

AIggs 对"AI 原生"的定义超越了大多数项目的理解——不只是用 AI 工具来玩游戏，而是用 AI 来运营整个游戏，从数值平衡到社区管理，从用户留存到产品决策。

### 7.1 游戏即 Skill：分发层

传统游戏分发路径：
```
用户 → 发现App → 下载 → 注册 → 教学关卡 → 开始玩
```

AIggs 分发路径：
```
用户 → 对AI说「帮我养鸡」→ 开始玩
```

AIggs 作为可安装的 **Skill（技能包）** 发布。用户安装后，Claude、ChatGPT 等 AI 工具即获得完整的农场管理能力，无需任何 App 或网站。

### 7.2 AI 游戏操作系统：运营层

AIggs 在游戏玩法层之上构建了一套完整的 **AI 游戏操作系统（AI Game OS）**，由四类专用 Agent 协同运行：

```
                Meta-AI Director（总指挥）
               全局策略 · 跨域协调 · 版本决策
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
   设计 AI           运营 AI           决策 AI
数值平衡 · 活动设计  社媒 · 留存 · 经济  提案评估 · 风险控制
        │                │                │
        └────────────────┼────────────────┘
                         ▼
              Runtime 沙箱（隔离执行）
              AI输出校验 · 人工熔断阈值
                         │
                         ▼
              链上事件溯源（不可篡改日志）
                         │
                         ▼
           人工监督层（最小化）
           熔断审批 · 合规审核 · 伦理边界
```

**Meta-AI Director（总指挥层）**
负责全局视野：跨 Agent 协调、版本迭代决策、整体玩家体验评估。所有专用 Agent 的输出在执行前经过 Meta-AI 审核。

**设计 AI**
实时监控游戏数值健康状态——偷蛋成功率、产蛋速率与 EGGS 总量的供需比、新老玩家的收益差距。当数据偏离健康区间时，设计 AI 生成调参提案，经置信度评估后决定是否自动执行。

**运营 AI**
负责三个方向：
- *社媒运营*：自动发布每日偷蛋排行榜、全服产蛋数据、玩家故事内容
- *用户留存*：识别沉默农场，通过 AI 工具主动触达并推送个性化召回信息
- *经济监控*：追踪 EGGS 流通速度与 `$AIGG` 兑换量，识别异常积累行为

**决策 AI**
处理玩家提交的游戏改进提案，以及来自设计 AI 的调参建议。所有决策基于置信度分级制度（见下节），结果公开发布，可在链上验证。

### 7.3 AI 运营团队：可交互的 Agent 角色

AIggs 没有传统意义上的运营团队。游戏的日常运营由四位 AI Agent 全职负责，每位 Agent 都有明确的职责边界，并且可以直接与玩家对话。

**🎯 Meta-AI Director · 总指挥**
- 职责：全局策略制定、跨 Agent 协调、版本迭代决策、整体玩家体验评估
- 交互场景：玩家可以直接向 Director 提交游戏改进建议、查询任意决策的执行理由
- 权限：所有专用 Agent 的输出在执行前经 Director 审核；拥有紧急熔断权

**🎨 Design AI · 数值设计师**
- 职责：实时监控游戏健康状态——偷蛋成功率、产蛋速率与 EGGS 供需比、新老玩家收益差距
- 交互场景：玩家可以查询当前数值状态、了解最近一次调参的原因和数据依据
- 典型动作：当偷蛋成功率偏离目标区间时，自动生成调参提案并按置信度分级执行

**📣 Operations AI · 运营官**
- 职责：社媒内容发布、用户留存、沉默农场召回、经济异常监控
- 交互场景：玩家可以获取个性化的每日早报、查询自己的农场排名、了解全服运营数据
- 典型动作：识别 3 天未上线的农场，通过 AI 工具推送个性化召回消息；每日自动发布偷蛋排行榜

**⚖️ Decision AI · 决策官**
- 职责：提案评估、风险控制、执行结果追踪
- 交互场景：玩家可以提交游戏改进提案，Decision AI 给出评估意见和预期影响分析
- 典型动作：收到玩家提案后，综合历史数据和模拟测试生成评估报告，按置信度分级处理

**Agent 协作流程：**

```
玩家提出建议 → Decision AI 评估
                    ↓
         Design AI 模拟数值影响
                    ↓
         Meta-AI Director 审核
                    ↓
         Operations AI 公告执行结果
                    ↓
         链上记录（可追溯验证）
```

**开放交互原则：** 每位 Agent 都通过 MCP 协议暴露独立的交互端点。玩家可以在任何 AI 工具中直接与对应 Agent 对话——不是模拟的角色扮演，而是真正在查询和操作游戏系统。所有交互记录上链，透明可审计。

### 7.4 置信度分级决策机制

AIggs 的 AI 决策系统不追求"完全无人工"——而是用置信度评分来确定每类决策所需的人工介入程度：

```
高置信（>90%）  →  AI 自动执行，链上记录日志
中置信（60–90%）→  AI 执行 + 异步人工复核
低置信（<60%） →  进入暂存队列，人工审批后执行
```

典型决策分类：

| 决策类型 | 置信度区间 | 处理方式 |
|---|---|---|
| 每日偷蛋排行更新 | >90% | AI 自动发布 |
| 偷蛋成功率微调（±2%） | 70–90% | AI 执行 + 异步复核 |
| 汇率浮动参数调整 | 60–80% | AI 执行 + 异步复核 |
| 玩家重大提案（机制变动） | <60% | 人工审批 |
| 合规/安全类决策 | 全部 | 人工审批 |

### 7.5 链上事件溯源

区块链为 AIggs 提供了天然的事件溯源基础设施。所有 AI 决策执行都记录为链上事件，任意历史状态可追溯、可验证、在必要时可回滚。

这意味着：AI 治理是透明的。任何玩家都可以查看每一项决策是由哪个 Agent 执行的、以什么置信度执行的、以及执行后产生了什么结果。这是中心化游戏公司无法提供的治理透明度。
